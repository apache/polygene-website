July 31, 2015

Apache Zest™ (Java Edition) Release 2.1

Release-notes

ZEST-40 - Rename “Qi4j” to “Apache Zest (Java Edition)”

Apache Zest Java Edition 2.1 is the first release of the Qi4j codebase under the Apache Software Foundation umbrella. To keep backward compatiblity, this version still use org.qi4j.* Java packages and Maven coordinates. Please note that this will change in 3.0.

Bug

  • ZEST-38 - Scheduler throws NPE
  • ZEST-41 - FileEntityStore doesn’t work on Windows
  • ZEST-55 - Type and naming collisions
  • ZEST-56 - Intermittent failures of ElasticSearch indexing on startup
  • ZEST-57 - ElasticSearch Query engine generate immense term queries on “large” associations
  • ZEST-58 - ElasticSearch do not support MatchesSpecification (regexp queries)
  • ZEST-59 - MixinsModels.primaryType() return wrong type on Java 8
  • ZEST-61 - In Lifecycle.create(), @This can’t be used in EntityBuilder’s to set non-Optional values.
  • ZEST-62 - If same Identity is used twice within the same UnitOfWork, it is not detected.
  • ZEST-64 - Extensions assemblies assemble config entities even if no config module is specified
  • ZEST-68 - Query DSL templateFor( .. ) fails to build template for ManyAssociations
  • ZEST-70 - ElasticSearch do not support NeSpecification
  • ZEST-73 - Redis EntityStore silently squash entities with an existing identity
  • ZEST-74 - SideEffects being executed more than once
  • ZEST-78 - File EntityStore silently squash entities with an existing identity
  • ZEST-107 - Entity SPI: JSONEntityState cloning logic is flawed

Improvement

  • ZEST-42 - Changes to Directory naming in FileEntityStore
  • ZEST-60 - Better error message when an Entity can not be found.
  • ZEST-65 - PassivationException should use Java 7 “suppressed exceptions” to gather all failures stacktraces

  • ZEST-67 - Make Activators injectables
  • ZEST-75 - Create/Update Entity from Value in Conversion Library
  • ZEST-77 - Adding ability to set MetaInfo on Module
  • ZEST-79 - Better error message when Mixin is missing.
  • ZEST-82 - Add possibility to get EntityReferences from Associations without loading the Entities
  • ZEST-83 - Remove serialization on ValueComposite creation
  • ZEST-89 - Introduce ModuleSpi as an injectable entry point
  • ZEST-90 - Better error message when an Entity type can not be found

New Feature

  • ZEST-22 - Add toValue() and toEntity() methods to the Qi4j API.
  • ZEST-47 - Complex Configuration Types
  • ZEST-63 - ApplicationBuilder for easier assembly with JSON support
  • ZEST-69 - NamedAssociations
  • ZEST-71 - Memcache based Cache Extension
  • ZEST-72 - Assembler adapters for common use cases (visibility, identity, configuration)
